# 週次分解スキル

評価シートの目標・行動計画を週次のアクション可能なタスクに分解し、**mdファイルとして出力**する。

## 発動トリガー

- 「週次に分解して」「週単位で計画立てて」
- 「今週やることを教えて」「今週の優先度は？」
- 「週次目標ファイル作って」「計画ファイル出力して」
- 評価シートをアップロードして計画を聞かれたとき

## ファイル構成

```
maclogger/
├── evaluation/
│   └── FY2025下半期_評価シート.xlsx  # 評価シート
├── weekly-plans/
│   ├── 2026-W04-plan.md              # 週次目標ファイル
│   ├── 2026-W05-plan.md
│   └── ...
├── reports/
│   ├── 2026-W04.md                   # maclogger週報
│   └── ...
└── reconcile/
    ├── 2026-W04-reconcile.md         # 突合結果
    └── ...
```

## 週次目標ファイルのフォーマット

**ファイル名**: `YYYY-WXX-plan.md`（例: `2026-W05-plan.md`）

```markdown
---
week: "2026-W05"
period: "2026-01-27 ~ 2026-02-02"
evaluation_period: "2025/09〜2026/02"
week_number: 22
total_weeks: 26
---
```

## タスク記法のルール

突合skillでパースしやすいように、以下の記法を守る：

```
- [ ] [カテゴリ] タスク内容 | 想定: Xh | 評価項目
```

### カテゴリ（必須）
- `[本業]` - プロダクト開発、クライアントワーク
- `[学習]` - 資格勉強、技術習得
- `[アウトプット]` - 記事執筆、ドキュメント作成
- `[改善]` - 自動化、ツール導入、プロセス改善

### 評価項目（必須）
- `G2` - 現在グレードの目標
- `G3` - 1つ上のグレードの目標
- `Be hungry` - 貪欲に学習・挑戦
- `Be an owner` - 締め切り・責任
- `Be sustainable` - 仕組み化・ナレッジ共有

複数の評価項目に該当する場合はカンマ区切り: `Be hungry, Be sustainable`

## ワークフロー

### Step 1: 評価シートの読み込み

評価シート（xlsx）から以下を抽出：
- 評価期間
- 能力評価の目標（G2, G3）
- バリュー評価の目標と行動計画
- 現在の振り返り内容（進捗状況）

### Step 2: 現在週の特定

```python
# ISO週番号で管理
from datetime import date
today = date.today()
week_number = today.isocalendar()[1]
year = today.isocalendar()[0]
week_id = f"{year}-W{week_number:02d}"
```

### Step 3: 週次タスクへの分解

**分解の原則**

1. **1週間で完了可能な粒度**
   - ❌ 「Rails学習を完了する」
   - ✅ 「Rails Tutorial Chapter 3を完了する」

2. **成果物が明確**
   - ❌ 「AWSの勉強をする」
   - ✅ 「AWS CLF模試を1回解いて、間違えた問題をまとめる」

3. **時間見積もりを含む**
   - 1タスク = 2〜8時間程度を目安

### Step 4: ファイル出力

```bash
# 出力先
output_path="maclogger/weekly-plans/2026-W05-plan.md"
```

ユーザーに出力先を確認してから書き出す。

## 週初め（月曜）の計画セッション

「今週の計画立てて」と言われたら：

1. **先週の計画ファイルを確認**
   - `weekly-plans/YYYY-WXX-plan.md` を読む
   - 未完了タスクを抽出

2. **先週の週報を確認**（あれば）
   - `reports/YYYY-WXX.md` を読む
   - 実際にやったことを確認

3. **今週の計画を作成**
   - 繰り越しタスクを含める
   - 新規タスクを追加
   - 優先度を調整

4. **ファイル出力**
   - `weekly-plans/YYYY-WXX-plan.md` を作成

## 評価期間全体の計画生成

「残りの週を全部計画して」と言われたら：

1. 残り週数を計算
2. 評価シートの未達成目標を抽出
3. 各週に割り振り
4. 複数の週次目標ファイルを一括生成

```bash
# 出力例
weekly-plans/
├── 2026-W05-plan.md
├── 2026-W06-plan.md
├── 2026-W07-plan.md
├── 2026-W08-plan.md
└── 2026-W09-plan.md
```

## Tips

### 現実的な計画を立てる

- 本業は予測不能な割り込みが多い → バッファを持つ
- 学習・アウトプットは週3-5時間が限界
- 「やれたらやる」タスクは別セクションに

### ファイル命名規則

- ISO週番号を使用: `2026-W05`
- 月曜始まり
- 年をまたぐ場合に注意（12月末〜1月初旬）
